> 更新时间：2026-03-07

## 一、项目目标

将**业务本体（Ontology）**融入 MILP 求解器的 Learning-to-Branch 流程，构建三层异构图：

```
本体层（业务实体）  ──桥接──  变量层  ↔  约束层
  员工、车辆、任务              x₁…xₙ      c₁…cₘ
```

用 **OntoGNN（三明治架构）** 学习分支决策，验证本体信息对求解效率的提升。  
目标投稿：NeurIPS / ICML 2026。

## 四、核心模块说明

### 4.1 生成器（src/generator/）

每个生成器输出两个文件：
- **`.json`**：包含业务实体节点（如 `Employee`、`Shift`）、属性、边关系
- **`.lp`**：标准 LP 格式，供 SCIP/Ecole 加载求解

Employee Scheduling 生成器特殊处理：原始约束矩阵是**全幺模（TU）**的（LP 松弛直接得整数解，无分支样本），已添加**加权技能约束**（junior=1/senior=2/expert=3）和**全局预算约束**打破 TU 性质。

### 4.2 图构建器（src/graph/builder.py）

```python
from graph.builder import load_and_build
data = load_and_build("path/to/instance.json", ecole_obs=obs)
# 节点类型: variable[V,19], constraint[C,5], Employee[E,5], Shift[S,5]（随问题类型变化）
# 边类型: variable↔constraint（数学层）, variable↔业务实体（桥接层）, 业务实体↔业务实体（语义层）
```

### 4.3 模型（src/model/ontognn.py）

```python
from model.ontognn import OntoGNN
model = OntoGNN(metadata, hidden_dim=64, use_ontology=True)
scores = model(data)  # shape: [num_variables]，值越高越优先分支
```

`use_ontology=False` 退化为纯数学 Baseline，用于消融实验对比。

### 4.4 训练流水线（src/train/）

```bash
# Step 1: 采集 Strong Branching 样本
python src/train/collect_data.py lp \
    --instance_dir data/raw/employee_scheduling \
    --output_dir data/processed/employee_scheduling \
    --disable_presolve

# Step 2: 训练 OntoGNN
python src/train/train_ontognn.py \
    --data_dir data/processed/employee_scheduling \
    --epochs 50 --use_ontology
```

## 六、关键踩坑记录

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| LP 实例全部根节点求解（0 分支样本） | Employee Scheduling 约束矩阵是全幺模（TU）的 | 添加加权技能+预算约束打破 TU；同时用 `--disable_presolve` 禁用 SCIP 预处理 |
| Infeasibility 级联 | `min_coverage` 需求超过总员工容量 | 改为 `randint(1,3)`，移除过紧的公平性/部门预算约束 |
| `builder.py` numpy 布尔 ValueError | `arr or other_arr` 在 numpy 数组上触发歧义 | 改为显式 `if x is None:` 判断 |
| Training Loss = NaN | Ecole `NodeBipartite` obs 中不适用特征为 NaN | 在 `collect_data.py` 中添加 `np.nan_to_num()` |
